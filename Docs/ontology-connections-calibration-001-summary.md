# Ontology Connections Calibration 001 Summary

## Purpose

Bucket the 18 reviewed connection entries to decide whether Connections should become a first-class ontology intake source.

## Answer

Yes: Connections should become a first-class ontology intake source, but not through the same flow as ordinary entries.

Connections are already compressed. They often arrive as beliefs, heuristics, process anchors, and pattern interrupts. The review flow should therefore start closer to candidate axiom review, with an explicit product-vs-personal boundary check.

## 1. Strong Candidate Personal Axioms

These are already stated like rules and appear central enough to enter a confirmed-review path once evidence is attached.

- **Delegation is three sentences**
  - If delegation includes what I want, how I think about it, and what done looks like, then handoff quality improves.
- **Use compute over cleverness**
  - If hand-crafted solutions will plateau, then use compute over cleverness.
- **Ten minutes exporting judgment compounds**
  - If I export judgment into a system, then that time compounds; if I only do the task, the value disappears.
- **Articulating something well changes relationship to it**
  - If I articulate something clearly, then my relationship to it changes because writing is thinking.
- **Capture first, structure later**
  - If I capture first and structure later, then I preserve information without blocking action.
- **You do not outwork the problem**
  - If I find the structural edge and build the simplest system around it, then results can compound without overwork.
- **Judgment strong, vocabulary gap**
  - If I struggle to specify what I want, then the problem is vocabulary, not judgment.

## 2. Product/System Principles

These are important, but they are about Understood or product design rather than Adam's personal behavior.

- **Products fight potential slipping away**
  - If a product captures otherwise-lost potential, then it creates meaningful lift.
- **Does this capture potential that would slip away?**
  - If a tool captures potential that would otherwise slip away, then it is worth attention.
- **My assignment is to create desire**
  - If my work creates desire, then the product has a stronger chance of becoming meaningful to users.

Recommended handling: keep visible as Connections for now, or route into a future product ontology lane.

## 3. Mixed Personal/Product Claims

These contain both personal operating principles and product/system implications.

- **My assignment is to create desire**
  - Personal: Adam's assignment is to create desire.
  - Product: the product must create desire to matter.
- **Does this capture potential that would slip away?**
  - Personal: Adam values capturing potential before it disappears.
  - Product: Understood should be evaluated by whether it captures otherwise-lost potential.

Recommended handling: split before ontology review. Do not review as one claim.

## 4. Remain Connection Only

These are useful prompts or anchors, but not yet strong enough as axioms without rewrite or evidence.

- **What is context without conviction? Regret.**
  - Powerful phrase, but needs a clearer if-then rule before review.
- **Energy moves forward, not backward.**
  - Useful identity anchor, but too compressed without examples.

Recommended handling: keep as Connections; use as prompts during future evidence review.

## 5. Needs Evidence Before Candidate Review

These are axiom-like but should be supported by lived examples before becoming high-priority candidates.

- **Anticipatory anxiety lies**
  - If anticipatory anxiety appears, then treat it as a feeling, not a fact.
- **Am I building a system or doing a task?**
  - If I am doing a task repeatedly, then I should ask whether I am really building a system.
- **Insights emerge through engagement**
  - If I engage directly with material, then insight emerges through action rather than separate analysis.
- **Work on what I can**
  - If I work on what I can control first, then uncertainty becomes easier to tolerate.
- **Momentum is the metric that matters**
  - If something creates momentum, then it is worth pursuing; if it creates stagnation, then it is worth avoiding.

Recommended handling: attach evidence from ordinary entries before confirmation review.

## Product Implication

Connections deserve a privileged ontology intake path.

Proposed future flow:

```text
connection
-> axiom-like?
-> personal/product/both?
-> candidate axiom draft
-> evidence links
-> human review
```

This is different from ordinary entries:

```text
ordinary entry
-> domain classification
-> product/personal boundary
-> claim splitting
-> evidence or candidate
```

Connections can skip some early extraction work because they are already compressed.

## Recommendation

Build a dedicated Connection calibration/review affordance later, but do not immediately promote all connections.

Start with the seven strong candidate personal axioms and attach evidence from existing entries before confirmation.
